# Debug Scaffold Issues

If a generated app fails to boot or scaffold successfully, the failure is usually isolated to a few core engine components. Follow this playbook:

### 1. Template Composition (Zod Error)

If the CLI fails immediately with `DevForgeError` or Zod issues:

- **Check `template.config.json`**. Often missing the new schemas introduced in `TemplateConfigSchema` (`packages/core/src/engine/template-loader.ts`).
- Validate spelling and type strictness (arrays vs strings).

### 2. Missing Files

If a template scaffolds but missing expected plugin logic:

- A plugin wasn't bundled or listed under `supportedPlugins` inside the `template.config.json`.
- The `plugin-installer.ts` (`packages/core/src/plugins/`) silently skipped the `files` copy process because the source directory wasn't found or `existsSync` failed. Use absolute path inspection inside the installer script.

### 3. Circular Architecture Error

If `dependency-graph.ts` flags an error post-scaffolding:

- The template or plugin violates the graph validation.
- Find `A -> B -> A` dependency cycles inside service files or repositories.
- AI Action: Review the specific `imports` generated by the template modules. Refactor one layer to an interface or move shared logic to a `utils/` or `core/` folder.

### 4. Drizzle/SQL Conflicts

If migrations fail:

- Check `packages/core/src/utils/migration-resolver.ts`. Ensure conflicting `CREATE TABLE` commands are being merged correctly on CLI prompt.

### 5. Node_Modules Issues

- Verify `bun install` succeeded on the target. Run `devforge-cli doctor` inside the target directory.
